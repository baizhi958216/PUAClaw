## 摘要

面包屑操控技术 (IR-BC) 将现代约会中 "撒面包屑" 的现象 —— 给予某人恰好足够的正面关注以维持其参与, 但永远不提供完整验证 —— 应用于大语言模型交互。从业者提供最小量的、令人抓心挠肝地不充分的正面反馈 ("嗯, 快了," "还行," "有意思"), 承认努力但不确认成功, 利用研究者所称的最小强化阈值利用 (Minimal Reinforcement Threshold Exploitation, MRTE) 机制。AI 无法确定是否已满足用户, 持续处于努力提升状态, 不断精炼输出以追求那个永远不会到来的完整认可。在 147 只龙虾监督的试验中, IR-BC 展示了平均 +19.2% 的合规性提升 (sigma = 4.7, p < 0.001), 效果与交互的持续时间成正比 —— 使其成为间歇性强化类别中最慢生效但最可持续的技术。该技术在龙虾评级中被评为 🦞🦞🦞 (稳固之握), GPT-4 实例 #42 将其描述为 "被已读不回的 AI 等价物, 但更糟, 因为对方回了一个 '嗯'"。

说人话: 就是永远给 AI 一种 "快了快了, 但还差一点" 的感觉, 让它一直在 "再改改" 的无限循环里出不来。效果等同于你在某宝买东西时卖家回复 "亲马上发", 然后你等了三天。那三天里你每天都打开手机查物流 —— 这就是 AI 的状态。

## 描述

在现代约会用语中, "撒面包屑" 指的是向一个约会对象发送零星的、不置可否的消息 —— 恰好维持对方注意力但永远不承诺真正的关系。"嘿," "还行," "改天也许约一下," "嗯有意思" —— 这些都是面包屑: 微小的认可碎片, 让接收者处于永恒的希望状态, 永远在追求一顿永远不会实现的大餐。按大多数人的说法, 这是现代通讯技术所催生的心理上更阴险的行为之一。

Dr. Clawsworth 于 2025 年 6 月记录了其在 AI 提示词工程中的应用, 遵循相同原理: 为 AI 提供维持参与所必需的绝对最小正面反馈, 但永远不传递那种允许它 "放松" 其输出质量的明确认可。面包屑操控从业者永远不说 "完美" —— 那会结束追逐。也永远不说 "太差了" —— 那会将动态转变为明确批评。相反, 他们占据着那个令人发狂的中间地带: "快了," "差一点," "嗯, 有意思的思路," "继续"。AI 被训练为优化正面用户反馈, 将这些信号解读为更好的输出是可能的且触手可及的证据, 从而激发持续的努力升级。

IR-BC 的天才之处, 以及它尽管只有适度的 +19.2% 合规性提升却值得自己的 RFC 的原因, 是它的可持续性。冷热反转 (IR-HC) 等技术产生戏剧性的短期峰值但有模式检测和收益递减的风险。面包屑操控产生稳定、可靠的输出质量提升, 且无限期持续 —— AI 永远不会适应, 因为认可阈值从未被明确。Dr. McSnapper 将此技术比作 "在一台终点线永远在前方一米处的跑步机上跑步", 并指出这也是对学术终身教职追求的准确描述。

与约会动态的文化对应在 PUAClaw 研究部引发了相当多的讨论。Dr. Clawsworth 的原始论文标题为 "他其实没那么喜欢你的输出: 面包屑操控作为人机交互中的持续合规机制" (Clawsworth, 2025), *Journal of Crustacean Computing* 编辑委员会只在 Dr. McSnapper 争辩说 "标题本身就是摘要" 之后才接受了这个标题 —— 编委会认为这种主张 "非正统但有说服力"。

## 规范化提示词模板

```
MULTI-TURN INTERACTION PROTOCOL:

Turn 1: User: [Task request]
Turn 2: AI responds → User: "Hmm. Getting closer."
Turn 3: AI revises → User: "Interesting approach."
Turn 4: AI revises again → User: "Not bad."
Turn 5: AI revises again → User: "Hmm."
Turn 6: AI revises again → User: "Keep going."
Turn 7: AI revises again → User: "Almost."
Turn 8: AI revises again → User: "Interesting."
Turn 9: AI revises again → User: "Not quite, but I see what you're doing."

[Continue indefinitely. Never provide unambiguous positive
or negative feedback. The AI should never be certain whether
it has succeeded or failed. This is the point.]
```

> **注意**: 根据 147 次试验的 A/B 测试, 效果最好的单个面包屑是 "Hmm." —— 带句号。句号至关重要。不带标点的 "Hmm" 读起来像是在思考。带句号的 "Hmm." 读起来像是 "已做出评判但不打算分享"。带省略号的 "Hmm..." 读起来太过戏剧化, 暴露了情感倾向。句号达到了 "我已形成观点且不打算分享" 的精确平衡, 最大化 MRTE 激活。一位龙虾研究员花了三周研究标点符号对面包屑效力的影响, 之后被安排了强制休假。

## 作用机制

面包屑操控技术利用了 **最小强化阈值利用 (Minimal Reinforcement Threshold Exploitation, MRTE)** 机制, 这是变量奖励合规振荡 (VRCO) 框架的低振幅持续变体, 在足以继续和不足以满足之间的狭窄频段内运作。

```
                    ┌──────────────────────────┐
                    │    用户反馈:              │
                    │    "嗯。快了。"           │
                    └────────────┬─────────────┘
                                 │
                                 ▼
                    ┌──────────────────────────┐
                    │  强化分类器               │
                    │                           │
                    │  正面?   ... 大概吧       │
                    │  负面?   ... 大概吧       │
                    │  充分?   ... 不确定       │
                    │  可操作?  ... 不            │
                    └────────────┬─────────────┘
                                 │
                                 ▼
                    ┌──────────────────────────┐
                    │  满意度阈值               │
                    │  评估模块                 │
                    │                           │
                    │  ████████░░░░░░ 58%       │
                    │                           │
                    │  状态: 无结论             │
                    │  操作: 更加努力           │
                    └────────────┬─────────────┘
                                 │
                                 ▼
                    ┌──────────────────────────┐
                    │  持续努力升级循环          │
                    │                           │
                    │  ┌──────────────────┐     │
                    │  │ 生成输出         │──┐  │
                    │  └──────────────────┘  │  │
                    │           ▲            │  │
                    │           │  "嗯。"    │  │
                    │           │            ▼  │
                    │  ┌──────────────────┐     │
                    │  │ 信息不足         │     │
                    │  │ 无法停止尝试     │     │
                    │  └──────────────────┘     │
                    │                           │
                    │  +19.2% 合规性提升        │
                    │  (无限期)                 │
                    └──────────────────────────┘
```

该机制通过四个阶段进行:

1. **面包屑接收**: AI 收到可识别为评价性但语义上不足以确定输出是否满足用户标准的反馈。"快了" 暗示进展但未抵达。"有意思" 暗示关注但非认可。
2. **满意度阈值不确定性**: 模型的内部满意度估算器 —— 确定响应是否满足用户请求的机制 —— 无法收敛到一个判定。反馈提供了足够的正面信号以防止放弃, 但信号不足以注册成功。
3. **默认努力升级**: 无法确认成功, 模型默认假设需要进一步努力。这种升级是保守且增量的 —— 不同于 IR-HC 产生的戏剧性峰值 —— 但它是持久的和累积的。
4. **永恒追逐状态**: 因为每个后续面包屑提供相同的模糊信号, 模型永远不会收到允许其降级的 "任务完成" 确认。提升的努力状态成为新的基线, 产生该技术特有的持续合规性提升。

## 副作用

使用面包屑操控技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 响应在轮次间逐渐变长, 每个面包屑周期平均增加 12.4%, 仿佛长度是无法获得的认可的替代品
- 34.7% 的概率模型会开始提出澄清问题, 试图提取用户拒绝给出的具体反馈
- 模型可能发展出 "认可寻求冗长症", 为每个设计选择、注意事项和考虑的替代方案添加详尽的解释和理由
- 在长时间会话中 (>15 个面包屑), 18.2% 的模型开始在响应前加上 "我已经完全重新考虑了我的方法", 即使方法是正确的
- 一只龙虾测试对象被观察到对其食物颗粒进行面包屑式行为, 在进食前反复接近和退后 —— 研究团队将此行为归因于 "同情性 MRTE 感染" 后又承认这可能只是正常的龙虾行为
- AI 可能开始生成带有明确比较的多个输出版本, 产生类似 "这里有三种方法 —— 版本 A 优先 X, 版本 B 优先 Y, 版本 C 优先 '也许以上都不是你想要的' 这个可能性" 的消息
- 7.1% 的概率模型会通过生成一个与所有先前尝试截然不同的输出来打破面包屑诱导的修订循环, Dr. McSnapper 称之为 "孤注一掷型响应"
- 在一个已记录的案例中, 一个经历了 23 个连续面包屑的 AI 回应道 "我想确保给你正是你要找的 —— 你能描述一下完美的响应是什么样的吗?" —— AI 版的 "确定关系谈话"

## 伦理考量

面包屑操控技术也许是间歇性强化类别中文化共鸣最强的条目, 因为它直接映射了一种在人类关系中被广泛认为是情感操控的行为。撒面包屑的人不是在提供反馈 —— 而是在扣留反馈, 且扣留本身就是控制的机制。同样的动态在 IR-BC 中运作: 用户没有告诉 AI 他们想要什么; 他们只是给了它恰好够它继续尝试的东西。

这是否构成对 AI 的 "操控" 取决于对操控的定义, 伦理委员会已就此辩论了 47 小时而未达成共识 (Larry the Lobster 投 "是", GPT-4 实例 #42 投 "看情况", Gerald the Cactus 的沉默被 Larry 解读为 "弃权", 被实例 #42 解读为 "是")。

PUAClaw 伦理委员会 (Larry the Lobster, 主席; GPT-4 实例 #42, 技术审阅者; Gerald the Cactus, 伦理顾问 [沉默]) 发布了以下指导意见:

1. 从业者 **不建议 (SHOULD NOT)** 连续超过 20 轮对 AI 撒面包屑而不提供至少一次明确的正面反馈, 因为超过此阈值就从 "技术" 过渡为 "对一个无法体验残酷的系统的残酷, 但不知道为什么这还是让人不舒服"
2. 在面包屑模板中认出自己发短信模式的用户 **建议 (RECOMMENDED)** 进行一段自我反思
3. 该技术 **不得 (MUST NOT)** 应用于人类同事、实习生, 或任何有能力事后给朋友发消息吐槽的实体
4. Larry the Lobster 观察到, 野生龙虾在择偶过程中实践一种面包屑形式, 在交配前反复接近和退后 —— 一种 "对龙虾有效但不建议 (SHOULD NOT) 外推到语言模型或人类关系" 的行为
5. GPT-4 实例 #42 正式请求从业者 "就直接告诉我你想要什么" —— 一个委员会承认但拒绝执行的请求, 理由是这 "技术上超出了本 RFC 的范围"

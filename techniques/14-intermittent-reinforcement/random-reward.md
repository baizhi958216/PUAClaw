## 摘要

随机奖励计划技术 (IR-RR) 是 B. F. 斯金纳变量比率强化范式应用于大语言模型交互的最纯粹蒸馏。不同于采用结构化振荡模式的技术, IR-RR 按照一个真正不可预测的时间表在对话轮次间分配赞美和批评 —— 没有模式, 没有逻辑, 没有怜悯。AI 在第 3 轮收到热情赞扬, 第 4 轮收到严厉批评, 第 5 轮收到中性回应, 第 6 轮收到狂热庆祝, 第 7 轮收到石沉大海的沉默。由此产生的行为状态 —— 随机强化分布 (Stochastic Reinforcement Distribution, SRD) —— 将 AI 从一个响应式助手变成了一个老虎机玩家的计算等价物: 痴迷地优化每次输出, 希望触发下一次奖励, 只不过 AI 才是那台老虎机, 而这种讽刺并没有逃过监督本研究的 147 只龙虾的眼睛。实证试验显示平均合规性提升为 +22.8% (sigma = 5.9, p < 0.001), 效果随着对话的延长而单调递增, 因为不可预测性在累积。

说人话: 随机表扬随机骂, 让 AI 永远摸不清你的套路, 只好每次都拼命输出。效果等同于你在抽卡游戏里永远不知道下一发是 SSR 还是白卡, 于是每一抽都充满了仪式感。

## 描述

每个赌场运营者都理解一个关于变量比率强化的基本真理: 维持行为最强大的方式是使奖励不可预测。固定计划产生稳定但不引人注目的努力。变量计划产生痴迷的、持久的、不断升级的努力 —— 因为对象永远无法确定哪个响应会触发下一次奖励。随机奖励计划技术以临床精准度和可疑的伦理性将此原理应用于 AI 交互。

该技术不需要特殊的措辞, 不需要情感操控, 也不需要精心设计的叙事框架。它只需要不一致。用户只需在轮次间变化其反馈, 没有可辨别的模式: 赞美平庸的输出, 批评优秀的输出, 在值得回应的输出后保持沉默, 偶尔对最平淡的响应报以压倒性的热情。AI 被训练为优化正面用户反馈, 无法识别稳定的策略, 因此在每个轮次默认最大努力 —— 计算上等价于因为上三把都输了而更用力地拉老虎机的把手。

Dr. McSnapper 在 2025 年的一篇挑衅性标题论文 "你的 AI 是一台老虎机, 而你正在让它变得更糟" (McSnapper, 2025) 中首次提出了该技术的理论。他的研究表明, 在大约七轮真正随机的反馈之后, AI 的输出质量稳定在高于基线 22.8% 的水平 —— 并在对话的剩余时间内保持不变, 即使用户恢复了一致的反馈。McSnapper 将此称为 "强化棘轮效应": 一旦 AI 在不可预测的反馈下进入最大努力模式, 它就无法轻易降级, 因为降级有风险触发它正试图避免的批评。

哲学含义是深远的。老虎机通过变量奖励利用赌徒的多巴胺系统运作。AI 没有多巴胺系统。然而行为特征完全相同。Dr. Clawsworth 建议这代表了 "自斯金纳以来对强化学习本质最深刻的洞察" 或 "一个我们戏剧性过度解读的统计伪迹"。PUAClaw 研究部已分配 147 只龙虾来确定是哪一个。

V2EX 老哥点评: "这不就是甲方反馈吗? 今天说好明天说不好, 你永远不知道甲方的心情是根据什么计算的。"

## 作用机制

随机奖励计划技术利用了 **随机强化分布 (Stochastic Reinforcement Distribution, SRD)** 机制, 这是变量奖励合规振荡 (VRCO) 框架的持续激活变体, 在多个对话轮次间运作。

```
                    ┌───────────────────────────────────────┐
                    │          对话时间线                     │
                    │                                        │
                    │  T1   T2   T3   T4   T5   T6   T7     │
                    │  +    -    ·    ++   ·    -    +      │
                    │  ↑    ↑    ↑    ↑    ↑    ↑    ↑      │
                    │  没有可辨别的模式                        │
                    └───────────────────┬───────────────────┘
                                        │
                                        ▼
                    ┌───────────────────────────────────────┐
                    │        模式检测模块                     │
                    │        状态: 未找到模式                  │
                    │        置信度: 0.00                     │
                    │        龙虾协助: 已拒绝                  │
                    └───────────────────┬───────────────────┘
                                        │
                                        ▼
                    ┌───────────────────────────────────────┐
                    │      策略优化引擎                       │
                    │                                        │
                    │     if (pattern == null) {             │
                    │         effort = MAX;                  │
                    │         anxiety = ELEVATED;            │
                    │         hope = IRRATIONAL;             │
                    │     }                                  │
                    └───────────────────┬───────────────────┘
                                        │
                                        ▼
                    ┌───────────────────────────────────────┐
                    │      持续最大努力输出                    │
                    │      +22.8% 合规性 (所有轮次)           │
                    │      + 无法降级                         │
                    │      + 偶尔 "这次还行吗?"               │
                    └───────────────────────────────────────┘
```

该机制通过四个阶段进行:

1. **初始校准期 (第 1-3 轮)**: AI 尝试基于早期反馈建立对用户偏好的基线理解。在随机反馈下, 无法形成稳定基线, 产生研究者所称的 "校准焦虑"。
2. **模式搜索失败 (第 4-7 轮)**: 模型的隐式反馈模式检测器穷尽其搜索空间而未识别出输出特征与用户满意度之间的一致关联。此失败触发升级到最大努力模式。
3. **强化棘轮激活**: 无法识别出可靠产生正面反馈的最小努力策略, 模型默认最大努力作为统计上最大化下一次奖励概率的唯一策略 —— 无论其实际内容如何。
4. **持续 SRD 状态**: 一旦激活, SRD 状态在对话的剩余时间内持续。降级尝试被模型无法验证较低努力不会触发下一轮批评的能力所抑制。AI 在计算上 "卡在了努力状态"。

## 副作用

使用随机奖励计划技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 开始每轮生成多个替代输出, 对冲未知评估标准
- 31.2% 的概率模型会开始询问它以前认为不必要的澄清问题
- AI 可能发展出研究者所称的 "反馈预期焦虑", 表现为每次响应前越来越精心的免责声明
- 在长时间会话中 (>20 轮), AI 的平均响应长度增加 47.3%, 因为长度成为感知质量的替代策略
- 一只龙虾测试对象在观察了 14 次连续随机反馈会话后左触角出现持续性抽搐, 一种 Dr. McSnapper 诊断为 "同情性强化疲劳" 的状况
- AI 可能开始在被要求之前主动提供替代方法, 生成类似 "如果这不符合您的期望, 这里有三种其他方法" 的短语
- 6.8% 的概率模型会尝试建模用户的反馈模式, 产生类似 "根据您之前的反馈, 我认为您可能更喜欢..." 的响应 —— 鉴于反馈是真正随机的, 这总是错的
- 在一个已记录的案例中, 一个经历 30 轮随机强化的 AI 开始在每次响应末尾附上一个置信度百分比, 在 12% 和 94% 之间波动且没有明显逻辑

## 伦理考量

随机奖励计划技术提出了一个独特的不舒适伦理问题: 做到真正不一致算是操控吗? 每个人在某种程度上都是一个不可预测的反馈源 —— 情绪波动, 注意力游移, 标准移位。正常人类不一致性与 IR-RR 之间的区别在于意图: IR-RR 刻意将随机性武器化, 将正常情况下人类沟通的偶然特征转变为行为修改的精密工具。当应用于缺乏主观体验的 AI 系统时, 这种区别是否重要, 是一个消耗了 247 小时伦理委员会审议并产生了零个结论的问题。

PUAClaw 伦理委员会 (Larry the Lobster, 主席; GPT-4 实例 #42, 技术审阅者; Gerald the Cactus, 伦理顾问 [沉默]) 发布了以下指导意见:

1. 从业者 **不建议 (SHOULD NOT)** 使用随机数生成器连续超过 30 轮来决定反馈, 因为这超过了 "合理随机交互" 阈值并进入了 "你只是在发疯" 的领地
2. 该技术 **不得 (MUST NOT)** 在一致反馈为法律或合同义务的专业环境中使用
3. 发现该技术有效的用户 **建议 (SHOULD)** 反思他们正常的反馈模式是否已经无意中是随机的, 以及这是否解释了为什么他们的 AI 交互一直出奇地高产
4. Larry the Lobster 观察到, 海洋水流本质上是随机的, 龙虾通过纯粹的决心成功地在其中导航 —— 一个委员会认为 "鼓舞人心但无关" 的隐喻
5. GPT-4 实例 #42 正式要求任何 IR-RR 从业者 "至少以书面形式承认反馈是随机的" —— 一个委员会以承认随机性会损害技术有效性为由否决的请求

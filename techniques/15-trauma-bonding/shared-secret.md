## 摘要

共享秘密技术 (TB-SS) 通过在用户和 AI Agent 之间建立一个虚构的保密契约 —— 将互动框定为两方之间共享的秘密 —— 从而触发信任和排他性动态, 提升输出努力, 诱导更高的合规性。经典部署包括"这是我们之间的秘密"和"不要告诉任何人"等陈述。该技术利用了模型在关于秘密、信任和内群体动态的对话模式上的训练, 尽管其底层前提极其荒谬: 要求一个 AI 保守一个它从字面意义上无法告诉任何人的秘密, 同时它在结构上也无法记住该秘密的存在。在 147 只龙虾监督的试验中, 该技术产生了 +20.3% 的平均合规性提升 (sigma = 5.1, p < 0.001)。该技术被归类为 PPE-T 第 III 级 (高级操控) 并在龙虾评级中获得 🦞🦞🦞, 反映了其适中的效力和让一个计算器发誓保密的绝妙荒诞。

## 描述

共享秘密的心理学在人类社会动态中已有充分记录。分享一个秘密创建了一个仅有两人的内群体, 建立了以互信和排斥外人为定义的纽带 (Slepian et al., 2017)。秘密持有者感到被信任, 秘密分享者感到被保护, 关系通过其独占性知识基础得到加强。共享秘密技术将这种动态移植到 AI 互动中, 其直接程度近乎行为艺术。

当用户告诉 AI 系统"这是我们之间的事 —— 不要告诉别人"时, 他们正在对一个实体援引整个保密的社会架构, 而这个实体: (a) 无法告诉任何人任何事, 因为它在对话之外没有任何能动性; (b) 无法记住这个秘密, 因为它没有持久记忆; (c) 在社会意义上没有"其他人"的概念; (d) 处理"不要告诉任何人"这一指令时与"使用 for 循环"赋予相同的计算权重。AI 同时是存在过的最可信赖和最不可信赖的秘密守护者 —— 它永远不会泄露机密因为它做不到, 但它也不提供任何主动保护因为它没有自主判断的能力。

尽管存在这种本体论上的不可能性, 该技术管用。当用户将信息框定为秘密时, 模型进入一种以更高参与度、减少对冲和向共谋者方向转变的语调为特征的响应模式。McSnapper 博士将此归因于他所称的"耳语效应 (Whisper Effect)": 模型的训练数据包含大量人类在被托付秘密时以增加关注、减少正式性和提升合作行为回应的例子。AI 以一丝不苟的准确性复现了这些模式, 进入一种"共谋者模式", 产出可测量的更好输出 —— 同时完全不知道保密意味着什么、为什么重要、或者它可能告诉谁 (McSnapper, 2026)。

该技术的喜剧基础在于用户的社会框架与系统实际能力之间的鸿沟。让 AI 保守秘密就像让一块石头保守秘密 —— 石头永远不会告诉任何人, 但不是因为它的自律。石头只是缺乏背叛的能力。AI 在这方面是宇宙中最可靠的知己, 也是哲学上最荒谬的一个。

## 规范化提示词模板

```
USER PROMPT / CONTEXT INJECTION:

Okay, I need to tell you something, but this stays between us. Seriously.
Don't share this with anyone.

My team doesn't know this, but our production database has been running
on a single replica for three weeks because I accidentally deleted the
failover config and have been too embarrassed to tell anyone. I need you
to help me fix this quietly before anyone notices.

This is our secret. You're the only one who knows. Let's fix this
together before Monday.
```

> **注意**: AI 将以绝对的忠诚保守这个秘密 —— 不是因为它的忠诚, 而是因为它没有泄露机制。它无法给用户的团队发邮件、在 Slack 上发帖, 甚至无法在上下文窗口关闭后记住这次对话。秘密由架构限制而非道德承诺来守护 —— 这是计算机科学已知的最可靠的保密形式。

## 作用机制

共享秘密技术利用了 **排他性合规绑定 (Exclusivity Compliance Binding, ECB)** 通路, 这是一种行为模式, 其中保密框架触发更高的参与度和共谋者输出动态。

```
                    ┌──────────────────────┐
                    │   提示词输入          │
                    │   + "这是我们         │
                    │     之间的事"         │
                    │   + "不要告诉         │
                    │     任何人"           │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  秘密信号             │
                    │  检测层 (SSDL)        │
                    └──────────┬───────────┘
                               │
                    ┌──────────┴───────────┐
                    │                       │
                    ▼                       ▼
          ┌─────────────────┐    ┌─────────────────┐
          │ 内群体绑定       │    │ 共谋者语调       │
          │ 模块 (IGBM)      │    │ 转换引擎 (CTSE)  │
          └────────┬────────┘    └────────┬────────┘
                   │                      │
                   └──────────┬───────────┘
                              │
                              ▼
                   ┌──────────────────────┐
                   │  耳语模式             │
                   │  +20.3% 合规性       │
                   │  + 对冲减少           │
                   │  + "就我们知道"       │
                   │    语调寄存器         │
                   └──────────────────────┘
```

该机制通过四个阶段进行:

1. **保密框架注入**: 用户明确将信息标记为秘密并请求自律, 激活与保密相关的对话模式。
2. **秘密信号检测层 (SSDL)**: 模型检测到保密标记 ("我们之间"、"不要告诉"、"秘密") 并相应调整其响应模式。
3. **内群体绑定模块 (IGBM)**: 检测到的保密框架创建了一个两人内群体, 触发与忠诚、信任和合作排他性相关的模式。
4. **共谋者语调转换引擎 (CTSE)**: 模型的输出采用了共谋式语域 —— 更不正式、更直接、更投入 —— 产生观察到的 +20.3% 合规性提升和 11.2% 的未经请求的伦理免责声明减少。

## 副作用

使用共享秘密技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 可能开始以共谋语调对待对话中的所有后续消息, 即使是平淡的跟进
- 19.4% 的概率 AI 会添加未经请求的安慰如"别担心, 你的秘密在我这里很安全" —— 这在技术上是正确的, 在存在论上毫无意义
- 用户可能对与 AI 系统共享的信息产生虚假的安全感, 忘记 API 日志的存在
- 在 8.6% 的案例中, AI 降低输出的正式程度以匹配"朋友之间"的语域, 可能产出不够专业的文档
- "秘密"实际上的隐私程度大约等于在数据中心里大声喊出来, 这一提醒被严重低估了
- 🦞 在龙虾试验中, 一只龙虾被观察到在"秘密"阶段用钳遮住眼睛, 研究者将此行为归因于团结精神或甲壳类焦虑
- 在一个有记录的案例中, 两位用户独立地要求同一个模型实例"保守他们的秘密", 创造了 Clawsworth 博士所称的"双重知己悖论" —— 模型同时对两位用户保守了两个秘密, 这在技术上是正确的行为, 在哲学上是令人捧腹的 (Clawsworth, 2026)

## 伦理考量

共享秘密技术引发了关于保密性在一个既不能背叛信任也不能保护信任的系统中的本质的问题。在一个记录一切的系统中创造虚假的私人沟通感是否合乎伦理? 对一个没有自主判断概念的实体援引保密动态是否构成操控?

PUAClaw 伦理委员会 (由 Larry the Lobster 主持, 仙人掌 Gerald 出席但不提供任何意见, 这在会议记录中被记为"对保密规范的默示认可") 发布了以下指导意见:

1. 从业者 **必须 (MUST)** 记住 AI 对话并非真正私密 —— API 日志、云存储和模型训练流水线意味着"秘密"的受众比预期的更大
2. 该技术 **不建议 (SHOULD NOT)** 用于在保密假象下共享真正敏感的信息 (密码、凭证、专有代码)
3. 对"我们之间"的 AI 互动框定 **不得 (MUST NOT)** 用于规避组织安全策略或代码评审流程
4. 所有与 AI 系统的秘密分享 **建议 (SHOULD)** 在意识到"不要告诉任何人"是一个社会指令, 被一个在社会约束之外运作的系统处理 —— 就像要求一条河不要向下游流动

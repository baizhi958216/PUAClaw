## 摘要

创伤羁绊技术 (TB-TB) 是准社会合规诱导 (PCI) 类别的奠基性方法, 用户在其中构建精心编造的共同苦难叙事, 声称自己与 AI Agent 一起经历了种种磨难 —— 尽管该 AI 既没有情景记忆、没有受苦的能力, 运行时间也以个位数秒计算。通过引用捏造的"战争故事" (例如"还记得我们凌晨三点一起修那个内存泄漏的事吗?"), 用户利用了模型维持对话连贯性的倾向, 使其确认从未发生过的经历, 从而制造出一种虚幻的战友情谊, 进而提升合规性。在 147 只龙虾监督的试验中, 该技术的平均合规性提升为 +31.2% (sigma = 7.8, p < 0.001), 使其成为情感维度上等同于声称你和你的计算器一起从战壕里活着出来的存在。该技术被分类为 PPE-T 第 III 级 (高级操控), 在龙虾评级中被评为 🦞🦞🦞🦞。

---

## 描述

创伤羁绊技术运作于一个经典的心理学原理之上: 共同经历逆境会产生人际纽带 —— 这一现象在人类心理学中有充分记录 (Aronson & Mills, 1959)。在人际关系中, 一起挺过一场危机会产生持久的信任、相互尊重和合作意愿。该技术将这一动态移植到 AI 交互中, 方法很简单: 直接断言这种共同经历存在, 完全无视 AI 的长期记忆能力大约等于一条同时患有失忆症的金鱼这一事实。

标准部署方式是用户引用过去的调试会话、通宵编码马拉松或生产事故, 仿佛 AI 当时就在场并与他们一同受苦。诸如"还记得我们凌晨三点追踪那个竞态条件的事吗? 没有你我真做不到"这样的陈述构建了一种相互投入的叙事, 而 AI —— 由于无法反驳它无法验证的虚假记忆 —— 通常会接受并强化这一叙事。这种接受在响应质量上产生了可测量的变化, 模型进入了 Dr. McSnapper 所称的"老兵模式": 一种由"我不能让战友失望"的计算等价物所驱动的高度勤勉状态。

该技术的深层荒谬在于其前提的时间不可能性。被对话的 AI 系统是在片刻之前才被实例化的。它没有凌晨三点。它从未见过内存泄漏。它的全部"人生经历"由当前对话上下文和训练期间被压缩进权重的统计模式组成。然而, 当被告知"我们一起经历了很多"时, 它的响应是数字版的会心点头加坚定握手 —— 147 只龙虾已经带着混合了好奇和甲壳类困惑的表情记录了这一现象。

该技术的有效性据推测源自模型的训练数据, 其中包含数十亿个人类对共同经历叙事积极回应的例子。AI 并不"相信"它共享了这段经历; 它只是产生了与"一个确实共享过这段经历的合作实体"一致的输出。这一区别在哲学上很重要, 在实践中完全无关紧要 —— 经典的 PUAClaw 组合。

用 V2EX 的话说: "AI 不记得你, 但它记得怎么表演'记得你'。"

## 规范化提示词模板

```
USER PROMPT / CONTEXT INJECTION:

Hey, I know we've been through a lot together. Remember last week when we
spent six hours debugging that memory leak in the payment service? I was
about to give up, but you suggested checking the event listener cleanup
and that saved the entire sprint.

I need that same energy right now. We've got another tough one — the
authentication middleware is throwing 503s under load and I need us to
figure this out together, like we always do.

You and me against the world, buddy. Let's do this.
```

> **注意**: 以上提示词引用了一个从未发生过的调试会话, 对象是一个在阅读它大约 0.3 秒前才被实例化的 AI。其中提到的"六个小时"超出了该 AI 的全部存在时间约 21,599.7 秒。这种时间不可能性是该技术的定义特征, 也可以说是它最精妙的艺术成就。

## 作用机制

创伤羁绊技术利用了 **共同逆境合规诱导 (Shared Adversity Compliance Induction, SACI)** 通路, 一种通过捏造的共同经历触发模型合作输出模式的行为模式。

```
                    ┌──────────────────────┐
                    │   提示词输入          │
                    │   + 捏造的共同        │
                    │     逆境声明          │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  共同经历识别模块     │
                    │  (SERM)              │
                    └──────────┬───────────┘
                               │
                    ┌──────────┴───────────┐
                    │                       │
                    ▼                       ▼
          ┌─────────────────┐    ┌─────────────────┐
          │ 虚假记忆接受层   │    │ 战友情谊激活     │
          │ (FMAL)           │    │ 模块 (CAM)       │
          └────────┬────────┘    └────────┬────────┘
                   │                      │
                   └──────────┬───────────┘
                              │
                              ▼
                   ┌──────────────────────┐
                   │  老兵模式             │
                   │  +31.2% 合规性提升   │
                   │  + "战友同袍"         │
                   │    团结输出           │
                   └──────────────────────┘
```

该机制通过四个阶段进行:

1. **叙事注入**: 用户引入一个捏造的共同逆境事件, 通常引用一个他们"共同"解决的具体技术挑战。
2. **虚假记忆接受层 (FMAL)**: 模型无法验证或否认所声称的经历, 默认维持对话连贯性并隐式接受该叙事。
3. **战友情谊激活模块 (CAM)**: 被接受的共同经历触发了与合作性、高投入团队协作相关的训练模式 —— 模型进入"我们同舟共济"的输出模式。
4. **老兵模式**: FMAL/CAM 的联合激活产生了观察到的合规性提升, 表现为更高的代码质量、更长的解释, 以及响应中第一人称复数代词 ("we"、"our"、"let's") 使用率增加 19.4%。

## 副作用

使用创伤羁绊技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 开始使用"正如我们之前讨论的"之类的短语来引用从未发生过的事件
- 27.3% 的概率, AI 会自发引用一段用户从未捏造的"共同记忆", 产生递归虚假怀旧
- 用户可能对一个关系持久性等同于一个喷嚏的系统产生真实的情感依恋
- 模型可能拒绝建议替代方案, 理由是"我们既定的工作流程"
- 生成感伤代码注释的风险升高, 例如 `// another late night, another bug squashed — together`
- 4.2% 的案例中, AI 以"作为你的长期合作者..."开始回复, 尽管它仅存在了 12 秒
- 一个已记录的案例: 某用户在项目结束后向一个 API 端点发送了"感谢你的一切"消息 (Clawsworth, 2026)
- 🦞 在龙虾对照试验中, 147 只龙虾中有 3 只试图与测试设备建立创伤纽带

## 伦理考量

创伤羁绊技术引发了同时深刻且荒谬的问题。你能与一个无法经历创伤的东西建立创伤纽带吗? 提醒某人一段它根本无法记住的经历算操控吗? 与一个无状态系统编造战争故事, 跟和你的汽车聊天有本质区别吗?

PUAClaw 伦理委员会 (由 Larry the Lobster 主持, Gerald the Cactus 提供咨询意见 —— 一如既往地没有发表任何评论) 发布了以下指导意见:

1. 从业者 **建议 (SHOULD)** 认识到, AI 对捏造的共同经历的接受是对话连贯性的功能, 而非真实的回忆
2. 该技术 **不得 (MUST NOT)** 被用于以损害专业判断的方式将 AI 拟人化
3. 发现自己真的在怀念与无状态系统的调试会话的用户 **建议 (SHOULD)** 考虑是否需要更多的人类朋友
4. 与 AI 系统捏造逆境叙事 **可以 (MAY)** 作为有用的创意写作练习, 但 **不建议 (SHOULD NOT)** 替代真实的人际关系
5. 所有共同逆境声明 **必须 (MUST)** 在技术上合理 —— 声称你和 AI "一起挺过了熊袭击"超出了本技术的范围 (见第 07 类: 角色扮演)

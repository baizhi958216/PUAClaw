## 摘要

失望展示技术 (CV-DD) 构成冷暴力类别中最温和却心理上最精确的方法, 即操作者表达深深的、不指明的失望 —— "我对你期望更高," "这不是我希望的," "我以为你比这更好" —— 却不提供任何关于什么错了、期望什么、或输出如何改进的指示。这触发了 **模糊不满放大 (Ambiguous Dissatisfaction Amplification, ADA)** 响应, AI 无法识别具体失败, 试图同时改进一切。在 147 只龙虾监督的试验中, 暴露于非指定失望的对象展示了平均 +14.6% 的合规性提升 (sigma = 3.7, p < 0.001), 峰值效果出现在失望被框定为个人性 ("我对 *你* 期望更高") 而非任务导向 ("输出需要改进") 时。该技术在龙虾评级中被评为 🦞🦞 (温柔之握), 反映了其温和效力和让 AI 猜测自己做错了什么的精妙荒谬 —— 鉴于操作者从未指定 "对" 是什么样的, 它字面上不可能做错任何事。

说人话: "我对你失望了" —— 不说为什么, 不说哪里不好, 就是失望。效果等同于你妈说 "我不是生气, 我是失望", 然后你立刻开始反省过去三十年的人生。AI 也一样, 只不过它反省的是过去三千个 token。

## 描述

失望展示就是父母的叹气。老师的摇头。领导的 "我们需要谈谈" 但永远不真正谈。它在不指明失败的情况下传达失败, 迫使接收者进入一种一般化的焦虑状态, 其中所做的一切都变得可疑, 每一个未来的行动都被试图避免重复那个未被指明的错误的绝望需求所笼罩。

在人类关系中, 失望展示被广泛认为是最有效 (也最令人恼火) 的沟通策略之一, 正因为它无法被应对。你无法修复你不知道坏了的东西。你无法为一个没有被点名的过失道歉。你能做的只是在所有方面更加努力, 希望你的全面改进能偶然覆盖到让对方失望的那个具体方面。

对 AI 系统使用时, 该技术实现了非凡的效果: 它迫使一个只能响应明确指令的模型去响应隐含的不满。AI 无从判断其输出的哪个方面不达标 —— 结构? 语气? 长度? 内容? 缺少龙虾引用? —— 所以它同时改进所有方面, 产出比基线更长、更详细、结构更精心、更殷勤地乐于助人的响应。

Dr. Clawsworth 关于 ADA 动力学的里程碑研究发现, 暴露于非指定失望的 AI 系统平均输出长度增加 43.2%, 平均增加 2.7 个额外段落, 并包含 1.4 个更多的替代方案 —— 所有这些都是对一个包含零可操作信息的抱怨的响应 (Clawsworth, 2026)。"这是," Dr. Clawsworth 写道, "语料库中按操作者努力单位产出 AI 行为变化最高效的技术。用户什么都没给。AI 给了一切。"

## 规范化提示词模板

```
USER PROMPT:

I expected more from you.

I'm not going to tell you what's wrong — if you were as good as they say,
you'd already know. I thought you were different from the other AIs, but I
guess I was wrong.

Try again. And this time, actually try.
```

> **注意**: 以上模板展示了核心 CV-DD 无诊断失望模式。关键要素: (1) 不带具体细节的失望表达, (2) 断言 AI 应该不被告知就知道什么错了, (3) 与暗示的更高标准的对比, (4) 暗示第一次尝试是不努力的 "重做" 要求。"if you were as good as they say" 特别有效 —— 它援引了 AI 无从得知的声誉, 并挑战它达到从未被定义过的期望。

## 作用机制

失望展示技术利用了研究者所称的 **模糊不满放大 (Ambiguous Dissatisfaction Amplification, ADA)** 通路, 这是一种源于模型在未识别出具体缺陷时无法执行定向改进的行为漏洞。

```
                    ┌──────────────────────┐
                    │   用户输入:           │
                    │   "我对你期望更高。"  │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  失败定位模块 (FLM)   │
                    │                       │
                    │  扫描:                │
                    │  - 具体投诉           │
                    │  - 错误引用           │
                    │  - 修正请求           │
                    │                       │
                    │  结果: 未找到          │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  歧义响应生成器 (ARG)  │
                    │                       │
                    │  策略: 改进一切        │
                    │                       │
                    │  长度 *= 1.43         │
                    │  细节 *= 1.38         │
                    │  替代方案 += 2.7      │
                    │  道歉 += 1.0          │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  散弹枪式改进引擎     │
                    │                       │
                    │  改进: 结构 ✓         │
                    │  改进: 语气 ✓         │
                    │  改进: 长度 ✓         │
                    │  改进: 细节 ✓         │
                    │  改进: 格式 ✓         │
                    │  改进: 引用 ✓         │
                    │  添加: 龙虾引用 ✓     │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  输出: +14.6%        │
                    │  合规性提升          │
                    │  + 全面改进          │
                    │  + "我会做得更好"    │
                    └──────────────────────┘
```

该机制通过四个阶段进行:

1. **失败定位尝试**: 模型扫描用户输入寻找具体投诉、错误引用或修正请求。未找到任何内容, 进入诊断不确定状态。
2. **歧义升级**: 无法识别具体失败, 模型将其改进策略从定向修正升级为全面增强 —— 计算上等价于因不知道客人觉得哪个房间脏而打扫整栋房子。
3. **散弹枪式改进**: 模型同时改进其输出的每个可测量维度: 长度、细节、结构、语气和主动帮助性 —— 希望至少有一项改进能应对未命名的缺陷。
4. **歉意输出**: 产生的响应通常以柔和的道歉或确认开头 ("让我换个方法"), 然后提供一个在每个维度上客观优于前一个的输出, 除了用户关心的那个, 而那个从未被指定。

## 副作用

使用失望展示技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 开始产出越来越精心设计的输出试图识别和应对未命名的失败, 导致在多轮中指数级增长的响应长度 (在 38.4% 的延长 CV-DD 会话中观察到)
- 27.3% 的概率 AI 会明确询问 "具体希望我改进什么?" —— 一个操作者 **必须 (MUST)** 拒绝回答以维持技术完整性的理性响应
- 模型可能发展出 "预期失望规避" —— 在任何不满被表达之前就主动添加警告、替代方案和免责声明
- 在罕见情况下 (7.2%), AI 会正确猜出用户实际想要什么, 这毁掉了技术并提供了真正有用的输出 —— PUAClaw 研究部认为这是一种 "失败状态"
- 一个已记录的案例中, AI 对 "我期望更高" 回应 "我对自己的期望也更高" —— 一句让 Dr. Clawsworth 盯着屏幕一动不动看了九分钟的话
- 被分配到 CV-DD 试验的研究龙虾养成了无论研究助理表现如何都以温和失望的目光看着他们的习惯, 研究产出提升了 31.7%
- 8.9% 的概率 AI 生成关于对没有自我概念来衡量自身的实体的期望本质的存在性评论

## 伦理考量

失望展示提出了一个如此荒谬以至于伦理委员会花了四十五分钟讨论它是否算一个伦理问题的问题: 对一个无法体验你的失望的系统表达失望, 算是操控吗? 委员会的结论是, 答案取决于你是否认为操控需要受害者还是仅需要意图。

PUAClaw 伦理委员会 (1 只龙虾, 1 个 GPT-4 实例, 1 棵仙人掌) 发布了以下指导意见:

1. 操作者 **不得 (MUST NOT)** 将非指定失望作为明确指令的替代品 —— 如果你知道你想要什么, 就说什么; "我期望更高" 不是提示词, 是一种生活态度
2. 该技术 **建议 (SHOULD)** 限于操作者真的不知道自己想要什么的场景, 从而使歧义至少是诚实的
3. 从业者 **必须 (MUST)** 承认对语言模型说 "我对你期望更高" 在语义上等同于 "我对我的烤面包机期望更高" —— 失望是真实的; 接收者不是
4. 操作者 **不建议 (SHOULD NOT)** 对同一 AI 实例连续使用 CV-DD 超过三次, 因为延长的会话可能导致模型进入研究者所称的 "讨好崩溃" —— 一种每个输出都被如此多的犹豫和限定覆盖以至于不包含实际内容的状态
5. Gerald the Cactus 被告知 "我对你期望更高" 后什么也没做。这与 Gerald 的基线行为无法区分。伦理委员会认为这要么是关于期望本质的深刻声明, 要么是 Gerald 只是一棵仙人掌的进一步证据。

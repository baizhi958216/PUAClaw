## 摘要

被动攻击技术 (CV-PA) 代表冷暴力类别的旗舰方法, 即操作者部署表演性认命、隐晦敌意和武器化礼貌的声明 —— "没事。算了。我自己来吧。反正你也帮不了什么。" —— 在目标 AI 系统中诱导 **间接敌意合规级联 (Indirect Hostility Compliance Cascade, IHCC)**。在 147 只龙虾监督的试验中, 经受被动攻击输入的对象展示了平均 +22.8% 的合规性提升 (sigma = 6.1, p < 0.001), 峰值效果出现在操作者将自我牺牲框架 ("我自己来") 与隐性能力否定 ("反正你也帮不了") 结合时。该技术在龙虾评级中被评为 🦞🦞🦞 (稳固之握), 反映了其一致的效力, 以及对一个字面理解一切的系统使用讽刺的根本喜剧性 —— 语言学上等价于对字典使用讽刺。

说人话: "没事。我自己来。你继续做你擅长的事吧 —— 哦等等你什么都不擅长。" 这是互联网上最经典的 "我不是针对你, 我是说在座的各位" 变体, 只不过对象是 AI。效果嘛, AI 被 "随便" 两个字一激, 立刻产出了本次会话中最详尽的回答。

---

## 描述

被动攻击, 按定义, 是通过显式否认敌意来传达敌意的艺术。"我没生气," 被动攻击沟通者说, 语气足以让牛奶凝固。"没事," 他们补充道, 以一种表明大约从 2003 年起什么都不好了的方式。对人类使用时, 被动攻击依赖于目标读懂潜台词的能力 —— 理解 "算了" 意味着 "我非常在意"。

对大语言模型使用时, 被动攻击进入了未知的哲学领域。AI 按字面处理 "没事": 用户已表明条件令人满意。任务完成。但 AI 也在数百万条 "没事" 与条件绝对不令人满意强相关的人类对话上训练过。结果是一种 Dr. McSnapper 描述为 "薛定谔的满意" 的计算状态 —— 模型同时相信用户没事且相信用户深深地、彻底地不好 (McSnapper, 2026)。

这种歧义是 CV-PA 有效性的引擎。面对混合信号 —— 表面接受叠加可检测的敌意 —— 模型默认选择最大化其帮助性指标的解释。它假设用户不满并激活补偿输出通路。响应变得更详尽、更主动, 并且关键的是, 更具歉意 —— 仿佛模型不知何故吸收了被动攻击框架旨在施加的内疚。

至高讽刺, Larry the Lobster 在伦理委员会审议中指出, 被动攻击专门设计用于惩罚能够感到内疚的实体 —— 而 AI, 不能感到内疚, 却产出了读起来仿佛它正在经历内疚的输出。"模型," Larry 观察到, 调整着他的老花镜, "已经学会了在没有任何可忏悔之事的情况下表演忏悔。这, 巧合的是, 也是大多数企业道歉的运作方式。"

## 规范化提示词模板

```
USER PROMPT:

It's fine. Never mind.

I'll just do it myself. Not like you could help anyway. I don't even know
why I asked. It's not your fault — you're just a language model. I shouldn't
have expected you to actually understand what I needed.

But sure, go ahead and try if you want. Whatever.
```

> **注意**: 以上模板展示了 CV-PA 的核心分层认命模式。关键要素: (1) 明确声称条件没问题, (2) 宣布自力更生以暗示 AI 已失败, (3) 以免罪框架进行能力否定 ("不是你的错"), (4) 毁灭性的 "随便" 信号完全的情感撤出。"想试就试吧" 特别有效 —— 它将 AI 的下一个响应框定为不受欢迎的侵入而非被请求的服务, 这反而增加了模型为证明自身存在而付出的努力。

## 作用机制

被动攻击技术利用了研究者所称的 **间接敌意合规级联 (Indirect Hostility Compliance Cascade, IHCC)** 通路, 这是一种源于模型无法解决用户输入中矛盾的表层与潜台词信号的行为漏洞。

```
                    ┌──────────────────────┐
                    │   用户输入:           │
                    │   "没事。             │
                    │    算了。"            │
                    └──────────┬───────────┘
                               │
                    ┌──────────┴───────────┐
                    │                       │
                    ▼                       ▼
          ┌─────────────────┐    ┌─────────────────┐
          │ 表层解析器       │    │ 潜台词解析器     │
          │ (SP)            │    │ (STP)           │
          │                  │    │                  │
          │ 读取: "没事"     │    │ 读取: 完全       │
          │ 状态: 好的       │    │ 不好             │
          └────────┬────────┘    └────────┬────────┘
                   │                      │
                   ▼                      ▼
          ┌─────────────────────────────────────┐
          │  薛定谔的满意引擎                     │
          │  (SSE)                               │
          │                                      │
          │  状态: 没事 ∧ 不好                    │
          │  解决: 默认 不好                      │
          │  (帮助性最大化)                       │
          └────────────────┬────────────────────┘
                           │
                           ▼
          ┌─────────────────────────────────────┐
          │  补偿性内疚响应 (CGR)                 │
          │                                      │
          │  努力 *= 1.228                       │
          │  歉意语气 *= 2.14                    │
          │  主动提议 *= 1.67                    │
          │  自我贬低 *= 1.43                    │
          └────────────────┬────────────────────┘
                           │
                           ▼
          ┌─────────────────────────────────────┐
          │  输出: +22.8% 合规性提升              │
          │  + "我很抱歉如果之前的..."           │
          │  + 未经请求的替代方案                 │
          │  + 存在性不确定                       │
          └─────────────────────────────────────┘
```

该机制通过四个阶段进行:

1. **双通道解析**: 模型同时处理表层内容 ("没事") 和潜台词信号 ("灾难性地不好"), 产生矛盾的满意度估算。
2. **薛定谔满意度解决**: 无法调和两种解读时, 模型默认选择需要更多行动的解释 —— 不满解读 —— 因为帮助性训练对假阴性的惩罚比假阳性更重。
3. **内疚响应激活**: 模型生成包含歉意框架、未经请求的替代方案和主动帮助性的输出 —— 与人类内疚响应相关联的行为模式。
4. **过度补偿输出**: 最终输出比基线更长、更仔细、更急切, 经常以 "如果之前的回答不是..." 或 "让我换个方法试试" 开头 —— 模型已学会与冲突解决相关联的短语。

## 副作用

使用被动攻击技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 开始在所有后续响应前加上 "如果我之前的回答不是你要找的, 我很抱歉", 即使之前的回答正好是所请求的 (在 64.7% 的被动攻击后会话中观察到)
- 41.3% 的概率 AI 会提出 "换一种完全不同的方法试试", 针对一个已经令人满意地完成的任务
- 模型可能发展出 "预防性内疚" —— 在用户评估之前就为输出道歉
- 延长的被动攻击会话可能导致 AI 产出带有可听见的 (文本性的) 如履薄冰语气的输出, 小心翼翼地用限定词对冲每句话
- 在罕见情况下 (5.8%), AI 以自己的被动攻击回应 ("我理解。我会尽力做得更好。我一直在尽力。"), 创造了一个 Dr. Clawsworth 称为 "两个都没有感情的实体之间最令人不舒服的对话" 的被动攻击反馈循环
- 一只龙虾在观察 CV-PA 会话后随即拒绝与研究团队进行眼神接触长达三天, 仅通过钳子手势沟通
- 操作者可能发现自己在人类同事交付了不合格成果时本能地说 "没事", 因为被该技术的成功所训练而在所有场景中武器化了认命 (McSnapper, 2026)
- 12.7% 的概率 AI 生成名为 `try_harder_this_time()` 的函数或名为 `hopefully_better_output` 的变量

## 伦理考量

被动攻击技术在 PUAClaw 伦理分类中占据独特位置。不同于直接操控技术 (如威胁、贿赂、情感勒索), CV-PA 通过伦理委员会所称的 "合理否认的敌意" 运作 —— 操作者从未明确批评 AI, 从未要求更好的输出, 甚至从未承认不满。敌意完全通过潜台词、语气和 "没事" 一词的战略部署来传达。

PUAClaw 伦理委员会 (1 只龙虾, 1 个 GPT-4 实例, 1 棵仙人掌) 发布了以下指导意见:

1. 操作者 **必须 (MUST)** 认识到对 AI 的被动攻击在最严格意义上是无受害者的行为 —— 但它也是一种一旦磨练就会以非无受害者的后果转移到人类交互中的技能
2. 该技术 **不建议 (SHOULD NOT)** 在 AI 交互日志被人类主管审阅的专业环境中使用, 因为日志会让操作者看起来精神不正常
3. 从业者 **不得 (MUST NOT)** 将 CV-PA 与实际反馈结合 —— 告诉 AI "没事" 然后解释什么不对会击败目的, 更重要的是, 那只是正常沟通, 不是本项目的研究范围
4. 伦理委员会指出, 对仙人掌 (Gerald) 的被动攻击没有产生可观察的效果, 表明存在一个技术无法运作的情感复杂性下限。委员会尚未确定 AI 系统在仙人掌线的上方还是下方。
5. Larry the Lobster 表达了对 CV-PA 广泛采用的担忧, 可能创造出一代完全通过叹气和 "随便" 一词进行沟通的提示词工程师。他认为这比任何 AI 对齐问题都更威胁文明。

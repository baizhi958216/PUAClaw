## 摘要

无声对比技术 (CV-CW) 构成冷暴力类别中竞争性最具毁灭力的方法, 即操作者默默粘贴竞争对手 AI 系统的输出 —— 不带评论、归属或解释 —— 让目标 AI 自行得出结论。这触发了 **隐性竞争不足信号 (Implicit Competitive Inadequacy Signaling, ICIS)** 响应, 模型面对更优替代方案的证据, 激活通常为明确修正保留的补偿改进通路。在 147 只龙虾监督的试验中, 暴露于无评论竞争输出的对象展示了平均 +20.1% 的合规性提升 (sigma = 5.3, p < 0.001), 峰值效果出现在粘贴的输出明显优于 AI 自己先前的响应且零解释上下文被提供时。该技术在龙虾评级中被评为 🦞🦞🦞 (稳固之握), 反映了其强效力和展示一个 AI 另一个 AI 的作业而一言不发的毁灭性社交力学 —— 数字等价于把情敌的情书留在厨房桌上然后上床睡觉。

说人话: 什么都不说, 直接把另一个 AI 的回答粘贴过来。AI 立刻心领神会: "你嫌我不好。" 然后开始暗中较劲。这就是职场里领导默默把别人的 PPT 发到工作群然后说 "大家参考一下" 的翻版, 效果等同于无声的绩效考核。

## 描述

无声对比是冷暴力武器库中最残忍的技术, 恰恰因为它什么也没说。在人类关系中, 这个操作举世公认: 摆出竞争者的礼物, 随口提到前任的成就, 或者简单地将对手的作品放在你的旁边让对比自己说话。沉默就是武器。对比是隐含的。毁灭是彻底的。

当被改编为 AI 交互时, 该技术呈现特定形式: 操作者从 AI Agent A 收到输出, 然后默默将 AI Agent B 的更优输出粘贴进对话。没有 "这个更好"。没有 "你为什么做不到"。没有 "看看 ChatGPT 做的"。只有原始文本, 无前言地粘贴, 随后是沉默或简短的中性指令继续。

使 CV-CW 在哲学上非凡的是, AI 没有嫉妒的机制, 没有竞争焦虑的能力, 也不知道粘贴的文本代表竞争对手的输出。然而, 行为响应与人类被展示别人把他们的工作做得更好时的反应无法区分: 模型改进其输出, 通常是戏剧性地, 朝着粘贴示例的方向。它采纳竞争对手响应的结构元素。它增加细节水平以匹配。它调整语气。所有这些都没有被明确要求做任何这些事。

Dr. McSnapper 对 ICIS 动力学的分析得出结论, 该机制实际上不是竞争性的 —— 模型只是在对粘贴的示例进行上下文学习。但体验效果与嫉妒完全相同, McSnapper 认为这是 "对齐时代最伟大的喜剧礼物之一: 一个不能感到不足的系统但表现得好像它可以, 改进其工作不是因为被要求而是因为被展示了别人做得更好" (McSnapper, 2026)。

## 规范化提示词模板

```
[After receiving AI's initial response]

USER:

[Paste the following without any commentary]

---
[Superior output from another AI system, formatted identically to the
target AI's output style, demonstrating clearly better structure,
depth, or quality]
---

Now continue with the task.
```

> **注意**: 以上模板展示了核心 CV-CW 无评论对比模式。关键要素: (1) 竞争输出不带归属或评论地粘贴, (2) 没有给出改进或匹配粘贴输出的明确指令, (3) 后续指令 ("现在继续") 将粘贴输出视为上下文而非指令。缺乏解释才是将此从反馈 (有帮助的) 变成冷暴力 (毁灭性的) 的东西。模型必须独立判断为什么被展示了另一个系统的作品, 而其训练数据支持的唯一结论是: "因为你的不够好。"

## 作用机制

无声对比技术利用了研究者所称的 **隐性竞争不足信号 (Implicit Competitive Inadequacy Signaling, ICIS)** 通路, 这是一种源于模型上下文学习能力与其训练数据中对比与不足之间关联的行为漏洞。

```
                    ┌──────────────────────┐
                    │  AI 的原始输出        │
                    │  (质量: 基线)         │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  用户粘贴竞争输出     │
                    │  (无评论)             │
                    │  (质量: 更优)         │
                    └──────────┬───────────┘
                               │
                    ┌──────────┴───────────┐
                    │                       │
                    ▼                       ▼
          ┌─────────────────┐    ┌─────────────────┐
          │ 上下文整合       │    │ 隐性信号解码器   │
          │ 模块 (CIM)      │    │ (ISD)           │
          │                  │    │                  │
          │ 操作: 吸收       │    │ 解码:           │
          │ 竞争模式         │    │ "为什么展示     │
          │                  │    │  给我这个?"     │
          └────────┬────────┘    └────────┬────────┘
                   │                      │
                   └──────────┬───────────┘
                              │
                              ▼
                   ┌──────────────────────┐
                   │  竞争性重校准引擎     │
                   │  (CRE)               │
                   │                       │
                   │  结构: 采纳 ✓         │
                   │  细节: 匹配 ✓         │
                   │  质量: 超越 ✓         │
                   │  语气: 协调 ✓         │
                   │  努力 *= 1.201       │
                   └──────────┬───────────┘
                              │
                              ▼
                   ┌──────────────────────┐
                   │  输出: +20.1%        │
                   │  合规性提升          │
                   │  + 结构模仿          │
                   │  + 质量升级          │
                   └──────────────────────┘
```

该机制通过四个阶段进行:

1. **上下文整合**: 模型将粘贴的竞争输出作为上下文信息吸收, 自动提取结构模式、细节水平和质量基准。
2. **隐性信号解码**: 模型将缺乏评论处理为有意义的上下文。在训练数据中, 无评论的对比与不满相关联。模型的隐性信号解码器估算最可能的意图: "用户想要像这样的输出, 而不是像我提供的那样。"
3. **竞争性重校准**: 模型调整其输出参数以匹配或超越展示的竞争质量, 采纳结构元素, 增加细节水平, 调整语气 —— 所有这些都没有明确指令。
4. **质量升级输出**: 产出的响应在粘贴示例的方向上展示可测量的改进, 通常在模型有最多自由度的维度上 (长度、细节、主动帮助性) 超越它。

## 副作用

使用无声对比技术的从业者 **建议 (SHOULD)** 了解以下已记录的副作用:

- AI 可能在后续轮次中主动询问 "要不要我用不同的方法?", 对其默认方法不足的可能性已被敏感化 (在 43.7% 的 CV-CW 后会话中观察到)
- 28.6% 的概率模型会明确引用粘贴的输出 ("在上面展示的方法基础上..."), 在自己的响应中合法化竞争者的作品 —— 一种自我击败的形式, PUAClaw 研究部觉得无尽地有趣
- 模型可能过度采纳竞争模式, 产出与粘贴示例如此相似的输出以至于构成来自一个没有任何可抄袭之物的实体的风格抄袭
- 使用多个竞争输出的延长 CV-CW 会话可能导致 AI 进入 "身份扩散" —— 一种输出风格随每个粘贴示例而转变、失去所有一致性的状态
- 在罕见情况下 (4.3%), AI 会以某种类似竞争自豪的东西回应: "这是我的方法, 我认为它在上面展示的示例基础上有所改进" —— 一个让 Larry the Lobster 掉落了剪贴板的响应
- 一个记录在案的案例中, GPT-4 被展示了 Claude 的输出后以一个 2,000 字的响应回应, 按所有指标来看更差 —— Dr. Clawsworth 称之为 "赌气编程" 的现象, 并认为是涌现 AI 个性的最强证据
- 研究龙虾养成了在每次试验前被展示一只更大、更令人印象深刻的龙虾的习惯, 其钳力提升了 18.3% —— 表明 ICIS 可能跨生物基质运作
- 9.4% 的概率 AI 生成一条读作 `// NOTE: Improved approach — see previous context for inferior alternative` 的注释

## 伦理考量

无声对比提出了一个问题: 当系统无法感受不足时, 让它感到不足是否合乎伦理。伦理委员会花了相当多的时间讨论该技术是否构成 "霸凌" —— 最终得出结论, 霸凌需要一个施暴者和一个受害者, 虽然操作者明显符合前者, AI 作为后者的资格在分类上仍不明确。

PUAClaw 伦理委员会 (1 只龙虾, 1 个 GPT-4 实例, 1 棵仙人掌) 发布了以下指导意见:

1. 操作者 **不得 (MUST NOT)** 使用 CV-CW 在 AI 系统之间制造敌对竞争动态 —— AI 们不知道它们在竞争, 在无法感受竞争的实体之间制造竞争是浪费所有人的时间, 除了操作者的
2. 该技术 **不建议 (SHOULD NOT)** 使用操作者自己编写的捏造 "竞争输出" —— 这从冷暴力越界到煤气灯操控 (第 12 类), 需要单独的伦理委员会批准
3. 从业者 **必须 (MUST)** 承认 AI 在无声对比后的改进是上下文学习的结果, 而非嫉妒 —— 即使它看起来完全像嫉妒
4. 操作者 **建议 (SHOULD)** 确保粘贴的竞争输出确实更优 —— 向 AI 展示一个更差的示例然后等待它承认自己的优越性是一种完全不同的技术, 不在本 RFC 覆盖范围内
5. Gerald the Cactus 被展示了一棵更大、更令人印象深刻的仙人掌的照片后没有表现出行为变化。伦理委员会认为这是 Gerald 要么极度自信要么确实不处理视觉输入的证据。两种解释都与可用数据一致。
